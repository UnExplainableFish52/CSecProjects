📖 Theory Speedrun (15 minutes)

What is a Firewall?

A firewall is a network security device that monitors and controls incoming and outgoing network traffic based on predetermined security rules.

Types of Firewalls

Type How It Works
Packet Filtering Inspects packets, allows/blocks based on IP, port, protocol
Stateful Inspection Tracks connection state, smarter decisions
Proxy Firewall Acts as intermediary, inspects application-layer data
Next-Gen (NGFW) Deep packet inspection + IPS + app awareness

Stateful vs Stateless

  • Stateless: Each packet judged independently (fast but dumb)
  • Stateful: Remembers connection context (slower but smart)

Firewall Zones

  • WAN: Wide Area Network (internet-facing, untrusted)
  • LAN: Local Area Network (internal, trusted)
  • DMZ: Demilitarized Zone (public-facing servers, semi-trusted)

📖 Understanding Firewall Rule Components

Component What It Does
Action Pass (allow), Block (deny silently), Reject (deny with response)
Interface Which interface traffic is coming FROM
Protocol TCP, UDP, ICMP, ANY
Source Where traffic originates (IP, network, any)
Destination Where traffic is going (IP, network, any)
Port Specific service port (22=SSH, 3389=RDP, 80=HTTP)

⚔️ Attack Scenario 1: Port Scanning (Reconnaissance)

Execute Attack from Kali:

# Full TCP SYN scan:
sudo nmap -sS -p- 192.168.1.100 -T4

# Aggressive scan with OS detection:
sudo nmap -A 192.168.1.100

# Stealthy scan (slow):
sudo nmap -sS -T2 192.168.1.100

Detection in Splunk:

# Search for multiple connection attempts from same source:
index=main source="pfsense" 
| stats count by src_ip dest_port 
| where count > 50
| sort - count

Detection in ELK/Kibana:

# Kibana Discover search:
source_ip: * AND destination_port: *

# Aggregation:
Terms on source_ip.keyword
Count > 50
✅ What to Look For: Single IP hitting 50+ different ports in short time = Port Scan

⚔️ Attack Scenario 2: Brute Force SSH

Execute Attack from Kali:

# First, allow SSH temporarily on pfSense for this test

# Install Hydra:
sudo apt install hydra -y

# Create password list:
echo -e "password\nadmin\n123456\npassword123" > passwords.txt

# Brute force:
hydra -l admin -P passwords.txt ssh://192.168.1.100

Detection in Splunk:

index=main source="pfsense" dest_port=22 
| stats count by src_ip 
| where count > 10
| eval attack_type="SSH Brute Force"

Incident Response Playbook:

- Log review shows suspicious activity

**STEP 2: VALIDATION**
- Confirm it's not false positive
- Check IP reputation (VirusTotal, AbuseIPDB)
- Analyze source and destination

**STEP 3: CONTAINMENT**
- Block malicious IP on firewall
- Isolate affected systems if needed
- Preserve evidence (save logs)

**STEP 4: ERADICATION**
- Remove malware if present
- Close exploited vulnerabilities
- Reset compromised credentials

**STEP 5: RECOVERY**
- Restore systems to normal operation
- Monitor for re-infection

**STEP 6: DOCUMENTATION**
- Write incident report
- Update threat intelligence
- Improve detection rules
